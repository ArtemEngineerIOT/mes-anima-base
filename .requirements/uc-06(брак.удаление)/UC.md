# Брак. Взвешивание

| Поле | Значение |
|------|----------|
| **Назначение** | Позволить оператору вносить данные о браке путём взвешивания |
| **Основной актор** | Оператор |
| **Второстепенные акторы** | Начальник смены, MES-система |

## Предусловия

1. Для каждой машины создана техническая учётная запись для ввода данных по взвешенному браку.
2. На ПК у весов запущен АРМ «Взвешивание брака» под технической учётной записью машины.
3. В системе настроен справочник «Коды событий».

## Основной поток событий (интеграция с весами)

1. Оператор приступает к взвешиванию брака. На отдельном ПК у каждых весов доступен АРМ «Взвешивание брака».
2. Оператор при необходимости выбирает машину (по умолчанию установлена машина, под учётной записью которой работает АРМ) и временной период для отображения этапов, выполненных на этой машине.
3. Оператор выбирает этап, по которому будет производиться взвешивание.
4. В правой половине экрана оператор выбирает весы (**IN** — входные, **OUT** — выходные).
5. Оператор производит взвешивание. Система автоматически подставляет вес в поле «Вес брака» напрямую с весов.
6. Оператор выбирает тип брака: нажатием «быстрых кнопок» (для 9 самых популярных видов брака) или выбором из выпадающего списка.
7. При необходимости оператор заполняет текстовое поле «Примечание».
8. Оператор нажимает кнопку «Зарегистрировать брак».
9. Система автоматически привязывает запись о браке к выбранному этапу (в составе заказа MES).
10. Данные по браку передаются в DWH.

## Альтернативные потоки

### А1: Без автоматических весов

Оператор вводит вес брака вручную в соответствующее поле, затем продолжается основной поток.

## Постусловия

Факт удалённого брака зафиксирован в MES. Данные доступны для построения отчётов и отображения в UI и передаются в 1С.
