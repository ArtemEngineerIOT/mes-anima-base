# Учет простоя оборудования

| Поле | Значение |
|------|----------|
| **Назначение** | Регистрация и классификация простоя оборудования оператором |
| **Основной актор** | Оператор |
| **Второстепенные акторы** | Начальник смены, начальник ОГП, технолог |

## Предусловия

1. Оборудование находится в режиме «Активно» в MES.
2. Оператор авторизован в системе на рабочем месте, привязанном к данному оборудованию.
3. В системе существует актуальный справочник кодов и причин простоев, синхронизированный с DWH.
4. Исполняемый заказ назначен на машину.

## Основной поток событий

1. В процессе работы происходит остановка оборудования (плановый осмотр, поломка, ожидание материала и т. д.).
2. Простой регистрируется автоматически при наличии данных об остановке машины: простой фиксируется, и система выводит оператору сообщение с необходимостью ввести данные (US-15).
3. MES отображает диалоговое окно с иерархическим справочником причин простоев (категория → подкатегория → конкретная причина).
4. Оператор выбирает соответствующую причину простоя (например: категория «Технический» → подкатегория «Поломка» → причина «Замена термопары»).
5. При необходимости оператор вводит текстовый комментарий.
6. Оператор нажимает «Сохранить описание».
7. MES меняет статус оборудования на «Простой», сохраняет запись о простое с привязкой к оборудованию, заказу (если был активен), оператору, причине и комментарию.

> MES должна фиксировать простои вне рамок выполнения заказа?

8. На экране начальника смены машина меняет статус на «Простой» (US-14).
9. Система автоматически отправляет уведомление ответственному при необходимости.
10. Оператор возобновляет работу. MES фиксирует время окончания, рассчитывает длительность простоя и возвращает статус оборудования «Готов к работе».
11. Пакет с описанием простоя отправляется DWH → 1С (US-17).
12. Обновляются следующие отчёты:
   - начальника ОГП по простоям (US-24);
   - начальника ОГП с трендами по простоям (US-18);
   - начальника смены ОГП — стандартный сменный отчёт (US-13).

> Будем обсуждать при проектировании раздела начальника.

## Альтернативные потоки

### А1: Остановка по окончанию заказа / отсутствию задания

1. На шаге 4 оператор выбирает причину «Организационный» → «Нет задания».

### А2: Простой запланирован (например, плановое ТО)

1. На шаге 4 оператор выбирает причину «Организационный» → «Плановый».

### А3: Ручное заведение простоя (администратор)

1. Администратор открывает раздел «Оборудование» → «Простои машин».
2. Выбирает машину из выпадающего списка.
3. Заполняет форму ввода полями «Время начала простоя» / «Время конца простоя».
4. Нажимает кнопку «Записать простой».

### А4: Добавление кода простоя (US-1)

> Будем обсуждать при проектировании раздела начальника.

1. На экране начальника ОГП пользователь нажимает кнопку «Редактировать коды простоя».
2. Открывается окно с таблицей кодов простоя (код | описание).
3. Нажимает кнопку «Добавить код».
4. Заполняет поля в появившейся строке.
5. Нажимает кнопку «Сохранить».
6. Код становится доступен операторам при учёте простоев.

### А5: Данные с машины не передаются

См. процесс **А3**.

### А6: Данные поступают очень часто («троит») (US-139)

1. В интерфейсе администратора предусмотрена настройка тайм-аута для объединения повторяющихся сигналов в группу.
2. Оператор регистрирует группу повторяющихся сигналов, а не каждый по отдельности.
3. При необходимости администратор может запросить «сырые» данные — увидеть состав группы сигналов (US-140).

### А7: Остановка машины вызвана браком

Начало альтернативного потока может быть вызвано наличием физической кнопки фиксации брака на пульте оператора.

Процесс выполняется аналогично основному, кроме пунктов:

- п. 4: оператор указывает причину простоя — «Брак»;
- п. 7: система производит регистрацию брака, а не простоя (заказ MES);
- п. 10: система фиксирует время окончания брака и создаёт запись в таблице «Брак. Регистрация → Зарегистрировать брак».

Далее см. UC-10, п. 6.

## Постусловия

1. В системе создана полная запись о простое: время начала / окончания, длительность, причина, ответственный оператор, комментарий, привязка к оборудованию и заказу.
2. Данные доступны для немедленного расчёта OEE и формирования сменных отчётов.
3. При необходимости заинтересованные стороны получили уведомление о простое.
