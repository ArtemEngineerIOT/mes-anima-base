# Обработка оперативных событий оборудования

| Поле | Значение |
|------|----------|
| **Назначение** | Обеспечить возможность оператору своевременно реагировать на события, поступающие от оборудования в реальном времени (например, удар ножа), и классифицировать их для корректного учёта выполнения производственного заказа и последующей аналитики. |
| **Основной актор** | Оператор участка |
| **Второстепенные акторы** | MES (интерфейс), система автоматического сбора данных с машин (PLC/датчики), начальник смены (контроль). |

## Предусловия

1. Оператор авторизован в МЕС на рабочем месте, привязанном к конкретной машине.
2. Производственный заказ активен, оборудование работает, данные с датчиков поступают в МЕС.
3. В системе настроен справочник причин событий (например, типы ударов ножа: брак, окончание настройки, смена заказа, плановая операция).

## Основной поток событий

1. Во время работы машины происходит событие — сигнал фиксируется датчиком и автоматически передаётся в МЕС.
2. MES регистрирует факт события: метку времени, текущий метраж, текущий рулон, идентификатор машины и т. д., и записывает данные в заказ MES.
3. В интерфейсе оператора (АРМ) появляется всплывающее уведомление или строка в блоке «Данные с машин» с требованием указать причину.
4. Оператор открывает форму обработки события. Система отображает предварительные данные (время, метраж, контекст заказа) и выпадающий список причин.
   - По каждому событию, необходимому для отображения на экране, необходимо предусмотреть объём данных по этому событию.
5. Оператор выбирает подходящую причину из списка (например, «Окончание настройки», «Брак», «Смена заказа»). При необходимости может добавить текстовый комментарий.
6. Оператор подтверждает выбор. Система обогащает запись о событии данными введённой причины; если событие не обработано, система блокирует выпуск продукта.
7. Событие считается обработанным и больше не требует внимания оператора. Данные используются для построения отчётов и контроля исполнения плана.

## Альтернативные потоки

### А1: Оператор не реагирует на событие в течение заданного времени

- Система может повторно напомнить о незаполненном событии (визуальный индикатор).
- Незаполненные события блокируют завершение заказа или рулона до указания причины.

### А2: Ошибочное событие (ложный сигнал)

- Оператор может выбрать причину «Ложное срабатывание», чтобы событие не искажало статистику.

### А3: Несколько событий подряд за короткий промежуток времени

- Система может группировать события в одно, если интервал между ними меньше настраиваемого порога (US-139), и запрашивать одну причину для всей группы.

### А4: Отсутствие автоматического сигнала с машины

- Оператор может вручную зафиксировать событие, если такая функция разрешена администратором системы.

## Постусловия

1. Каждое событие зафиксировано в MES с полным набором атрибутов.
2. Данные доступны для последующего анализа выполнения производственного плана и оценки эффективности работы оборудования.
