# UC-12 — Материалы. Заказ со склада

**Статус:** Согласовано

## Назначение

Обеспечить возможность оператору сформировать заявку на доставку материалов (сырья, полуфабрикатов, упаковки) к производственной машине/машинам в соответствии с утверждённым производственным планом по этапам.

## Акторы

| Роль | Актор |
|------|--------|
| Основной | Оператор ОГП |
| Второстепенные | MES (интерфейс, взаимодействие с DWH), 1С (обработка заказа, подбор рулонов, складской учёт), начальник смены (консультации при внештатных ситуациях), склад (приём и исполнение заявки) |

## Предусловия

1. Оператор аутентифицирован в MES.
2. В системе актуальны данные производственного плана (этапы, привязанные к машинам).
3. В хранилище данных (DWH) доступны и регулярно обновляются:
   - ресурсные спецификации этапов;
   - складские остатки с признаками блокировок, початости, сроков годности, локаций.
4. Для каждой машины в DWH доступен справочник «Доступные склады для машин» (склады возврата, выпуска, значения по умолчанию), синхронизированный с 1С. Виды номенклатуры определяются непосредственно из ресурсных спецификаций. Данная информация является справочной; средством обмена данными двух систем служат JSON-пакеты.

## Макет экрана «Заказ материалов»

Экран — **двухколоночный** layout без общих фильтров в шапке страницы.

| Колонка | Заголовок | Назначение |
|---------|-----------|------------|
| Левая | **Формирование заказа** | Выбор этапов APS, сводный заказ, отправка заявки |
| Правая | **Статус заказа** | Частичная доступность, локация у машины, блокировки FR |

Референс-макет: `assets/Frame_5795-…png` (двухпанельный экран оператора).

### Левая колонка — «Формирование заказа»

1. **Контекст машины** — строка вида «Вы заказываете к машине — {код машины}» (машина оператора / машина текущего контекста заказа).
2. **Таблица этапов APS** — поиск; колонки: выбор, **этап**, дата заказа, клиент, продукт, количество, старт, завершение. Кнопка **«Сформировать заказ»**.
3. **Блок «Заказ материалов»** — поиск; таблица: номенклатура, запрошенное количество (редактируемое), единицы измерения; удаление строки.
4. **Галочка «Смена материалов»** — признак для 1С.
5. **Тумблер «Заказать конкретные рулоны/серии»** — при полуфабрикатах в заказе; таблица RemainsOut: выбор, номенклатура, серия, количество, ед. изм., срок годности. Кнопка **«Добавить в заказ»**.
6. **Футер заявки** — «Ко времени», «Комментарий для склада»; кнопки **«Отправить заявку»** (активна после успешного «Добавить в заказ») и **«Очистить заявку»**.

### Правая колонка — «Статус заказа»

1. **Плашка частичной доступности** (при ошибке 1С) — текст «Для номенклатуры X доступно только Y из Z…» + сводная таблица доступных позиций.
2. **Подблок «Локация у машины»**:
   - фильтры **машина** и **вид номенклатуры** (мультиселект);
   - таблица: выбор, машина, номенклатура, вид номенклатуры, серия, количество, ед. изм., **статус**, **FR**;
   - статусы: «Заказано», «Доступно», «Заблокирован» (подсветка строки);
   - кнопки **«Печать этикетки»** и **«Обновить»**.
3. **Подблок «Причины блокировки»** (А3):
   - блокируемая номенклатура (read-only по выбранным строкам);
   - выбор причины FR (список из DWH);
   - комментарий для склада;
   - кнопка **«Передать блокировку»**.

## Основной поток событий

### 1. Переход на экран

Оператор переходит на экран **«Заказ материалов»** (отдельный пункт главного меню).

### 2. Выбор этапов плана

В левой колонке **«Формирование заказа»** отображается таблица **этапов производственного плана** для машины заказа (данные APS). Каждая строка содержит чекбокс.

Оператор отмечает один или несколько этапов, для которых требуется заказать материалы.

### 3. Формирование заказа

Оператор нажимает **«Сформировать заказ»**. MES запрашивает DWH: ресурсные спецификации по выбранным этапам.

### 4. Сводная таблица материалов

В блоке **«Заказ материалов»** формируется сводная таблица.

| Столбец | Описание |
|---------|----------|
| Номенклатура | Одинаковая номенклатура из разных этапов **объединяется**; только **основные** позиции (альтернативы сырья не выводятся) |
| Запрошенное количество | Редактируемое поле |
| Единица измерения | кг, рул и т.д. |

**Условия отбора (DWH):**

- номенклатуры из спецификации;
- склады из **DepartmentWrhOut** для машины заказа;
- назначение = текущему этапу (**только полуфабрикаты**).

**Дополнительно:** галочка **«Смена материалов»**, датапикер **«Ко времени»**, **«Комментарий для склада»**, удаление строки.

### 5. Заказ конкретных рулонов/серий

При наличии **полуфабрикатов** (только сырьё — тумблер неактивен) доступен тумблер **«Заказать конкретные рулоны/серии»**.

При включении — таблица **RemainsOut** с чекбоксами. Условия отбора рулонов — как в п. 4 + номер этапа полуфабриката = текущему этапу.

### 6. Редактирование заявки

Корректировка количеств, время доставки, комментарий, удаление строк.

### 7. Добавление в заказ и отправка

1. **«Добавить в заказ»** — проверки пакета и доступности 1С.
2. При успехе разблокируется **«Отправить заявку»**.
3. Оператор нажимает **«Отправить заявку»**.

### 8. JSON-запрос в 1С

- тип заказа (сырьё / полуфабрикат / упаковка / готовая продукция);
- по номенклатуре: код, количество, ед. изм.;
- признак **«Смена материалов»**;
- выбранные серии (если тумблер включён);
- время доставки, комментарий.

### 9–10. Ответ 1С

- **Успех:** документ «Заказ на перемещение»; в **«Локация у машины»** — статус **«Заказано»**.
- **Частичная доступность:** уведомление + таблица в правой колонке; повторить п. 4–9.

### 11. Локация у машины

Актуальный перечень на складе машины (или общего склада нескольких машин).

Фильтры (мультиселект): **машина**, **вид номенклатуры**.

Обновление: **«Обновить»** или автоинтервал (настройка системы) из DWH.

| Статус | Описание |
|--------|----------|
| Заказано | Ожидание доставки |
| Доступно | Материал доставлен и отсканирован |
| Заблокирован | Серия с FR (А3) |

### 12. Завершение

Оператор может вернуться на экран для нового заказа или контроля доставки.

## Альтернативные потоки

### А1. Ошибка валидации / нет связи с 1С

Ошибка от 1С или **«Сервис 1С недоступен»**; повтор или эскалация.

### А2. Полуфабрикаты на финишных стадиях (Паучи, Бэги, Резка)

При тумблере «конкретные рулоны» — все доступные рулоны номенклатуры (не только текущий заказ); 1С — FIFO.

### А3. Блокировка серии

1. Выбор FR в **«Причины блокировки»**.
2. Выбор серий в таблице локации.
3. Комментарий FR (редактируемый).
4. **«Передать блокировку»** → колонка FR, JSON в 1С.

## Постусловия

1. В 1С — документ «Заказ на перемещение».
2. В MES — позиции **«Заказано»** в локации у машины.
3. Оператор проинформирован.
4. После доставки — **«Доступно»**.
5. При ошибке — корректировка и повтор.
