# Брак. Регистрация

| Поле | Значение |
|------|----------|
| **Назначение** | Позволить оператору фиксировать возникновение и окончание брака в процессе работы машины (в том числе без её остановки) с возможностью последующего указания типа дефекта для обеспечения прослеживаемости. |
| **Основной актор** | Оператор |
| **Второстепенные акторы** | Система сбора данных с датчиков (PLC), MES-система, начальник смены |

## Предусловия

1. Оператор аутентифицирован в системе.
2. Заказ запущен, машина работает в штатном режиме, система получает данные о текущем метраже.
3. В системе настроен справочник типов брака.

## Основной поток событий

1. Оператор обнаруживает дефект.
2. Оператор устраняет его или вкидывает флаги для устранения на последующих этапах.
3. Оператор открывает блок «Брак. Регистрация».
4. Нажимает кнопку «Добавить строку» в блоке зарегистрировать брак.
5. В интерфейсе оператора появляется необработанное событие (запись в таблице) с информацией о метраже дефекта.
6. Оператор в таблице выбирает незаполненную строку.
7. Оператор последовательно заполняет строку, указывая: начало брака, окончание брака, тип дефекта, цвета флагов, вброшенных в рулон, статус брака, рулон (автоматически подставляется активный, но есть возможность выбрать любой).
8. Оператор нажимает кнопку «Зарегистрировать брак». Строка записывается в БД и запись публикуется в системе.
9. Если оператор пытается закрыть заказ или выпустить рулон, не обработав все события, система выводит предупреждение и блокирует завершение до тех пор, пока все типы дефектов не будут указаны.

## Альтернативные потоки

### А1: Фиксация брака на ходу

1. Оператор обнаруживает дефект, который можно устранить на ходу, без остановки машины.
2. Оператор открывает блок «Брак. Регистрация».
3. Оператор нажимает кнопку в интерфейсе MES «Начало брака».
4. Система фиксирует метку времени и текущее значение счётчика метров (метр начала дефекта).
5. Оператор проводит регулировку оборудования для устранения дефекта.
6. После того как дефект устранён, оператор нажимает кнопку «Окончание дефекта».
7. Система фиксирует время и текущий метраж как метр окончания дефекта, автоматически вычисляя длину бракованного участка.
8. В таблице блока «Зарегистрировать брак» создаётся новая строка с зафиксированным началом и окончанием брака.
9. Оператор переходит к пункту 7 основного потока без указания начала и конца брака — они уже заполнены автоматически.

### А2: Отмена регистрации дефекта

Если оператор случайно нажал кнопку начала, он может завершить регистрацию дефекта, нажав «Окончание брака», после чего удалить запись из таблицы дефектов как ошибочно введённую с помощью кнопки «Удалить запись».

### А3: Ввод типа дефекта сразу после окончания

Оператор может сразу после нажатия «Окончание дефекта» заполнить отсутствующие данные по вновь созданной строке брака.

### А4: Невозможность определить метр начала / окончания (например, сбой датчика)

В случае если сигналов о метраже с машины нет, нажатие кнопок приведёт к созданию строки со значениями начало / окончание = NULL. Система позволит оператору ввести метраж вручную.

## Постусловия

В системе зафиксирован факт брака с точными метрами, типом дефекта и привязкой к текущему заказу и рулону. Данные доступны для отчётов по браку и для передачи на последующие стадии (резка / перемотка) как рекомендация по вырезке.
