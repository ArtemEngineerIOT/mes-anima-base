# UC-15 — Выпуск

## Назначение

Регистрация выпущенных рулонов (полуфабрикатов и готовой продукции), печать этикеток.

## Акторы

- **Основной актор**: оператор ОГП
- **Второстепенные акторы**: MES (интерфейс), 1С (учёт), начальник смены

## Предусловия

1. Производственный заказ активен.
2. В MES доступны данные о спецификации и параметрах выпуска.
3. В DWH доступны настройки складов для каждой машины/участка.
4. В DWH доступен справочник серий из 1С для проверки уникальности генерируемых серий.

## Основной поток событий

1. Оператор открывает блок **«Выпуск»**.
2. Система отображает форму выпуска для текущего заказа/этапа.
3. При начале намотки система генерирует идентификатор серии по шаблону: 6 цифр номера этапа + пробел + порядковый номер рулона. Пример: `123456 1`. Этот идентификатор используется для привязки всех событий по рулону: брак, прослеживаемость и другие связанные события.
4. MES проверяет уникальность серии по справочнику серий из 1С, выгруженному в DWH.
   - В 1С уникальность серий поддерживается.
   - MES сверяется со справочником DWH при генерации новых серий.
   - Если из-за редкой ошибки возникает дублирование номера заказа/этапа, 1С вернёт ошибку при первой отправке.
5. Оператор вводит данные о выпущенном рулоне:
   - для полуфабрикатов: вес нетто (совпадает с брутто) и метраж; целевой источник метража — датчик машины;
   - для готовой продукции — будет описано отдельным UC, пока не рассматривается;
   - при необходимости устанавливает флаг **«Требуется перемотка»**; флаг может быть установлен на любом участке при выпуске полуфабриката, если потребуется дальнейшая перемотка;
   - при необходимости устанавливает флаг **«Составной рулон»**;
   - указывает выходную номенклатуру, если спецификацией предусмотрен выпуск нескольких номенклатур; по умолчанию выбрана основная номенклатура, но оператор может переключить на **«Отходы»**, если они предусмотрены;
   - указывает склад назначения: промежуточный или склад готовой продукции, разрешённый для машины/участка.
6. В таблице **«Рулоны на размотке»** оператор выбирает один или несколько исходных рулонов, из которых изготовлен выпускаемый рулон, и нажимает **«Привязать»**.
7. Оператор нажимает кнопку **«Зарегистрировать выпуск»**.
8. MES сохраняет данные о выпуске в заказе MES и отправляет их в 1С через веб-сервис в формате JSON. Каждая серия регистрируется последовательно. Для полуфабрикатов можно считать правилом: рулон = палета; для готовой продукции это правило не применяется и будет описано отдельно.
9. После успешного сохранения в 1С MES дополняет данные для этикетки со штрихкодом и информацией о рулоне и отправляет этикетку на печать. При необходимости оператор может перепечатать этикетку позже через отдельную кнопку. Для каждого типа номенклатуры используется своя этикетка.
10. До подтверждения перемещения рулона на склад оператор может редактировать данные выпуска. После подтверждения склада редактирование блокируется (**US-96**).
11. После нажатия кнопки **«Зарегистрировать выпуск»** изменение выпущенной серии блокируется.
12. Система хранит историю данных, необходимых для генерации этикеток. По запросу пользователя с соответствующими правами может быть сформирована и распечатана копия этикетки (**US-136**).

## Альтернативные потоки

### A1 — Автоматическое заполнение веса для полуфабрикатов

При вводе в одно поле (вес нетто) второе автоматически дублируется.

### A2 — Выпуск составного рулона (Jumbo roll)

Оператор указывает, что рулон содержит несколько заказов, и вводит соответствующие данные по каждому заказу.

### A3 — Выпуск с признаком «Требуется выдержка» (ламинация)

После выпуска полуфабриката система фиксирует дату/время готовности к следующей операции и не допускает планирование ранее этого срока.

### A4 — Фиксация связи с исходными сырьевыми рулонами

MES автоматически подставляет в выпадающий список текущий рулон (единицу входного материала). Оператор может выбрать из списка другой при необходимости.

### A5 — Блокировка выпускного рулона (**US-148**)

1. В выпадающем списке с поиском **«Причины блокировки»** оператор выбирает FR блокировки для блокируемой серии/серий. Справочник причин передаётся в MES через DWH.
2. Оператор отмечает одну или несколько серий в таблице склада у машины, проставляя галочки.
3. В поле **«Комментарий для склада»** автоматически подставляется комментарий FR с возможностью изменения.
4. Оператор при необходимости вносит правки в комментарий.
5. Оператор нажимает кнопку **«Передать блокировку»**.
6. В строке серии в колонке **«Статус NCCS (FR)»** отображается номер выбранного FR.
7. Пакет с заблокированной серией передаётся в 1С в формате JSON.

## Постусловия

1. Рулон зарегистрирован в системе.
2. Этикетка напечатана.
3. Данные переданы в 1С, остатки на складе готовой продукции увеличены.
4. История выпуска доступна для просмотра.

## Вопросы

1. Необходимо добавить и детализировать реализацию прослеживаемости в блоке **«Выпуск»**.
